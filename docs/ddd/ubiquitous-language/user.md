# ユーザー系

## ユーザー（User）

### 定義
- サービスを利用する主体

### 説明
- 認証とデータ分離の単位

## ログイン（Login）

### 定義
- ユーザーとして操作できる状態になること

### 説明
- 認証に成功し、利用が許可された状態

## ログアウト（Logout）

### 定義
- 認証状態を終了し、ユーザーとして操作できない状態にすること

### 説明
- 利用中の認証状態を無効化する

## ユーザー名（UserName）

### 定義
- ユーザーが設定する表示名

### 説明
- サービス内でユーザーを識別・表示するための名称

## メールアドレス（EmailAddress）

### 定義
- 認証に使用するユーザーのメールアドレス

### 説明
- ログインやメール認証で利用する

### ルール
- ユーザー間で一意

## パスワード（Password）

### 定義
- ユーザーが認証に利用する秘密情報

### 説明
- 登録・ログイン時に入力される

### ルール
- 平文のまま永続化しない

## パスワードハッシュ（PasswordHash）

### 定義
- パスワードをハッシュ化した保存形式

### 説明
- 認証時はパスワードと比較する

### ルール
- 平文パスワードからのみ生成する

## メール認証（EmailVerification）

### 定義
- メールアドレスの所有を確認する手続き

### 説明
- 認証に成功するとユーザーは認証済みになる

### ルール
- 認証済みになった時点で認証日時を記録する

## メール認証トークン（EmailVerificationToken）

### 定義
- メール認証に使用する一時トークン

### 説明
- ユーザーに紐づく
- 有効期限と消費状態を持つ

### ルール
- 1回のみ使用できる
